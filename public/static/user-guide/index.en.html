BUDA User Guide

This User Guide is designed to help you navigate and make best use of BDRC’s library platform BUDA. It is a comprehensive guide to searching and accessing the texts and data in our vast library. As the largest digital archive of Tibetan and Buddhist literature in the world, we have more than 64,000 volumes digitized and more than 31 million pages made freely available to the public. We also have a treasure trove of Person and Place data, as well as an ever-growing corpus of e-texts. Please read on to learn how to find a text, how to download it, how to search for its author’s other works, and much more.

How to Find a Text

BDRC's archives contain millions of pages of texts, and when paired with the powerful search engine and library platform BUDA, it is possible to rapidly identify the text you are looking for and discover new ones you were unaware of.

Before searching for a text on BUDA, it is important to take a moment to learn how texts are classified in order to understand the context of the results.

Understanding Works and Versions

As our users know well, BDRC’s massive archive contains tens of thousands of individual texts, and for 15,000 of these texts our archive contains multiple distinct versions. For instance, BDRC has over 30 unique versions of the 12th-century classic Dakpo Targyen (dwags po thar rgyan), or Jewel Ornament of Liberation, by Gampopa. For the Dakpo Targyen, BDRC possesses multiple distinct versions of each of these four formats. To say that BDRC has 34 versions of the Dakpo Thargyen creates a distinction between 1) the Dakpo Thargyen as an idea and 2) specific representations of the Dakpo Thargyen.

On BUDA, the former is the work and the latter are versions. Gampopa’s masterpiece thus exists as an abstract entity (work) and also as a physical or digital text (version) that has been reproduced and edited many times over the last 800 years and for which many old copies are still extant.

When you start your search on BUDA, the search is already set to searching “in versions.” The results of this default version search include all available versions of a work in our database. Searching for a work, on the other hand, results in a single record for each abstract work, which includes a list of available versions in its work record.

Searching for a Text

Every search on BUDA starts the same: by choosing a search facet, entering keywords, and selecting a language.

1. In the search bar, select in versions from the drop down menu.

2. Type the name of the text you want to find. As you type, BUDA will narrow the language choices.

3. After you finish typing, select the language and input format you're using to perform the search.

Or enter a text’s version RID (resource ID) and choose one of these options from the language input drop down list:

  • Find Resource With This RID allows you to enter a text’s version RID, such as MW26071_0026, and go directly to its record.
  • Find Associated Resources returns resources associated with an RID.

Searching for a text

Note: If you select the language before you finish typing, you can continue on the results page.

Sorting the Results

Depending on the text you searched for and how you spelled it, there might be many results. You could scroll through the results to find the text you're looking for, but BUDA can sort the results in various ways to speed your search.

Note: The asset availability filters default to Access Possible and Catalog Only in version searches. These settings show only materials with fully accessible texts and images.

Search results

Below the search bar are two drop down menus.

The left one sorts results by popularity, closest matches, title, and year of publication. There is also an option to reverse the order of any of these sortations.

The right one determines how tightly matches are defined.

Sorting options

Let's look at the left one first.

To sort the results:

1. Open the drop down menu.

2. Choose the filter that best suits your search.

  • Popularity, the default setting, sorts the results starting from the work with the most associated records on BUDA to the work with the fewest associated records.
  • Closest Matches sorts the results by closest match in terms of textual distance between the searched name and the matched name.
  • Title sorts the results alphabetically.
  • Year of Publication sorts the results from oldest publication date to newest.

3. Select Reverse Order to reverse the order of a search and sort from least popular to most popular, for example.

Left sorting options

Now for the drop down menu on the right.

1. Open the drop down menu.

2. Choose the filter that best suits your search.

  • Partial or exact match, the default setting, returns results are exact matches, as well as those that contain the search terms anywhere within it, such as “rdzogs pa chen po” when searching for “rdzogs chen.”
  • Exact match only returns results that contain the complete search terms as well as longer text strings that contain the search terms within them.
  • Exact full match only returns results that exactly match the search terms.

Right sorting options

Refining the Results

BUDA's search filters offer a powerful way of refining your search.

  • Exact Match provides the same functionality as the filter below the search bar.
  • Version Type allows you to filter for standalone versions of texts or ones that are part of a larger collection, enabling you to find “kun bzang bla ma’i zhal lung” as a standalone text or as part of a sungbum, for example.
  • Associated Tradition contains filters for various traditions, depending on the search results.
  • Asset Availability includes options to only include results for works that BUDA has scans or e-texts for.
  • Match Context contains options to filter the results by whether the search terms appear in the title field of a work record or in its summary.
  • Genre contains filter options for commentaries, liturgies, sutras, and more, depending on the search results.
  • Topic allows you to filter the results by topics like Buddhist studies, medicine, etc., depending on the search results.
  • Language contains filters to sort the results based on the language of the text.
  • Script enables you to sort the results based on the character sets that are represented in the version and its metadata. This is helpful if you are researching the Ume script, or only want to see texts in Uchen, for example.
  • In Version contains filters to sort the results by the name of the version. Select Show all to see the rest of the options in this list.
  • Author allows you to filter the results by author. Select Show all to see the rest of the author choices.
  • Print Method includes options to filter the results based on the physical characteristics of the original item. This filter is ideal if you're working with specific printing techniques, or if you prefer modern publications.
  • In Collection contains filters to sort the results based on the collection a version is in.
  • Provider contains options to filter the results by providers, such as 84,000 or Leiden University, for example. Links to external providers’ records for a work can be found on the work’s record page.

To refine the results, select the options that best match your search.

Search filters

Tip 1: You can also exclude results by selecting the X in the red square that appears when you hover over a filter.

Excluding results

Tip 2: If you’re looking for e-texts or scans, open the Asset Availability filter and select Has Scans or Has E-text to filter for either of those.

How to Find Data About a Person

BUDA's person records provide a wealth of biographical data about the authors contained in our archives, as well as links to their writings and texts about them.

Searching for a Person

Every search on BUDA starts the same: by choosing a search facet, entering keywords, and selecting a language.

1. In the search bar, select in persons from the drop down menu.

2. Type the name of the person. As you type, BUDA will narrow the language choices.

3. After you finish typing, select the language and input format you're using to start the search.

Or enter a person’s RID (resource ID) or a significant year in their life and choose one of these options from the language input drop down list:

  • Find Associated Resources returns resources associated with an RID.

Navigating a Person Record

A person record contains information about a given person, including events in their life, people and places they are associated with, and their written works.

A person record

At the bottom of a person page, is a gray box labeled Related Works. This contains all the works associated with this person. These associated items are divided into two groups: Created By and About. Created By lists the works written by the person. About lists the works regarding the person.

Searching for a Person Within Texts

Another valuable source of information about persons is the BDRC e-text corpus, which includes many biographies and histories among its 1+ million pages. Search for personal names within e-texts by selecting in E-texts in the main search bar.

How to Find a Place

Place records on BUDA contain precise GIS coordinates of the place on the map where available, links to Wikidata, links to places contained within it and places it is contained in, as well as relevant information such as name variants, place type, associated tradition, and people affiliated with and texts about or printed at the place. About one third of all place records in BUDA contain precise GIS coordinates and are mapped.

Using the Image Viewer Interface

The viewing experience on BUDA can be even better than reading physical texts. Thanks to the platform's image processing options, you can not only zoom in on high-definition images but also use built-in image manipulation tools to adjust the brightness, contrast, and saturation, or even invert the colors. This makes it easier to read texts that have low contrast between letters and paper, have text bleeding through from the other side of the pecha, or have been smudged, for example.

To learn how to find a scan of text, go here.

This section of the guide points out BUDA's viewer features so that you can get the best viewing experience.

Using the E-text Viewer

The majority of e-texts available on BUDA were generated using optical character recognition (OCR) technology, rather than manual input. Although OCR for Tibetan script is not completely accurate, it is precise enough to enhance the discoverability and usefulness of the BDRC archive. It is best used in combination with scanned images, which can be viewed in tandem by following the steps below.

This section of the user guide describes how to use the e-text viewer.

  • Toggle the Monlam Grand Tibetan Dictionary on and off.

Using the Monlam Dictionary

The Monlam Grand Tibetan Dictionary comprises 200,000 entries in 223 volumes and is the most authoritative Tibetan language dictionary. Having it inside the etext view makes it easy to look up words and gain a richer appreciation of a text's meaning.

Opening the Dictionary

The dictionary works in both unicode Tibetan and Wylie. To open it:

  • Select Monlam Dictionary or begging bowl icon, or check the circle beside it.
  • Highlight a word.
  • Select Look up in Monlam Dictionary.

Using the Dictionary

With the dictionary open, you can continue highlighting words to look up their definitions.

Tip: While using the dictionary, you can copy and paste by using CMD/CONTROL+C.

Closing the Dictionary

Toggle off the dictionary in the menu bar or select X in the dictionary panel to close it.

Opening and closing the dictionary

Opening Scans

Opening scans is useful for when you want to check the accuracy of an OCRed e-text.

There are two ways of opening scans:

  • To open all of a text's scans, select Show scans on the lower menu bar.
  • To open a single scan, select the image icon or image number above the e-text page.
